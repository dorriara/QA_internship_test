# Задание 2.1 – Баг-репорт

**ID**: BR1  
**Краткое описание**: Нельзя создать объявление без поля statistics  
**Шаги воспроизведения**: 1. Отправка POST /api/1/item c JSON без поля statistics
```json
{
    "sellerID": 111121,
    "name": "Test Item",
    "price": 100
}
```  
**Ожидаемый результат**: HTTP 200 или 201 и сообщение с ID  
**Фактический результат**: HTTP 400  
___

**ID**: BR2

**Шаги воспроизведения**: 1. Отправка POST /api/1/item c JSON с некорректным sellerID
```json
{
    "sellerID": 11111,
    "name": "Test Item",
    "price": 100,
    "statistics": {
        "likes": 10,
        "viewCount": 15,
        "contacts": 3
    }

}
```

**Фактический результат**: HTTP 200

**ID**: BR8  
**Краткое описание**: Отсутствует валидация на диапазон значений 111111-999999 для sellerID при получении объявлений по идентификатору продавца  
**Шаги воспроизведения**: 1. Отправка GET /api/1/:sellerID/item c некорректным sellerID «11111»  
**Ожидаемый результат**: HTTP 400 и сообщение о некорректном sellerID  
**Фактический результат**: HTTP 200  
___

**ID**: BR9  
**Краткое описание**: Неправильный HTTP-код при получении статистики по некорректному ID для API v.2  
**Шаги воспроизведения**: 1. Отправка GET /api/2/statistic/:id с некорректным ID «123123»  
**Ожидаемый результат**: HTTP 400 и сообщение о некорректном ID  
**Фактический результат**: HTTP 404 и сообщение о некорректном ID  
___
